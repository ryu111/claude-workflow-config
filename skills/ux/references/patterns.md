# UX Patterns

常見 UX 模式與最佳實踐。

## 導航設計

```
主導航結構：
├── 全域導航（頂部/側邊）- 主要功能入口
├── 區域導航（次級選單）- 功能內部切換
├── 麵包屑（路徑顯示）- 知道目前位置
└── 快速操作（FAB/快捷鍵）- 常用功能
```

### 原則

| 規則 | 說明 |
|------|------|
| 層級 ≤ 3 | 避免迷路 |
| 當前位置標示 | 用戶知道在哪 |
| 3 次點擊到達 | 重要功能易達 |

## 表單設計

### 結構

```
┌─────────────────────────────────┐
│ 標題：清楚說明表單目的           │
├─────────────────────────────────┤
│ ┌─────────────────────────────┐ │
│ │ 標籤（在輸入框上方）        │ │
│ │ [輸入框：placeholder 示範]  │ │
│ │ 輔助文字（格式說明）        │ │
│ └─────────────────────────────┘ │
│                                 │
│ ┌─────────────────────────────┐ │
│ │ 必填欄位 *                  │ │
│ │ [輸入框]                    │ │
│ │ ❌ 錯誤訊息（紅色）          │ │
│ └─────────────────────────────┘ │
│                                 │
│ [次要按鈕]      [主要按鈕 →]    │
└─────────────────────────────────┘
```

### Checklist

- [ ] 標籤在輸入框上方（不是 placeholder）
- [ ] 必填欄位有標示
- [ ] 即時驗證，不等提交
- [ ] 錯誤訊息在欄位旁邊
- [ ] Tab 順序合理
- [ ] Enter 可提交

## 搜尋與篩選

```
[🔍 搜尋框（自動完成）]
     ↓
[即時搜尋建議]
     ↓
[搜尋結果] + [篩選器] + [排序]
     ↓
[無結果時：建議 + 相關內容]
```

### 無結果狀態

```
┌─────────────────────────────────┐
│         🔍                       │
│                                 │
│   找不到「關鍵字」的結果         │
│                                 │
│   建議：                         │
│   • 檢查拼字                     │
│   • 使用較少關鍵字               │
│   • 嘗試相關詞彙                 │
│                                 │
│   [瀏覽所有商品]                 │
└─────────────────────────────────┘
```

## 載入狀態

| 等待時間 | 處理方式 |
|----------|----------|
| < 100ms | 不需要指示 |
| 100ms - 1s | 簡單 spinner |
| 1s - 10s | 進度指示 + 說明 |
| > 10s | 進度條 + 預估時間 + 取消選項 |

## 空狀態設計

```

│     你還沒有任何專案             │
│                                 │
│  建立第一個專案開始使用          │
│                                 │

```

**原則**：空狀態不是空白，而是引導。

## Modal vs Drawer vs Inline

| 模式 | 適用情境 |
|------|----------|
| **Modal** | 需要專注、重要決策、確認操作 |
| **Drawer** | 詳細資訊、設定面板、次要流程 |
| **Inline** | 快速編輯、簡單操作 |

## 確認對話框

破壞性操作使用確認對話框：

```
┌─────────────────────────────────┐
│ ⚠️ 刪除專案                      │
├─────────────────────────────────┤
│ 確定要刪除「我的專案」嗎？       │
│ 此操作無法復原。                 │
├─────────────────────────────────┤
│ [取消]           [刪除] (紅色)   │
└─────────────────────────────────┘
```

**原則**：
- 說明後果
- 破壞性按鈕用警告色
- 預設選中安全選項

## 錯誤訊息

### 原則

1. 說明發生了什麼
2. 解釋為什麼發生
3. 提供如何解決

### 範例

```
❌ "錯誤：無效輸入"

✅ "電話號碼格式不正確。請輸入 10 位數字，例如：0912345678"
```

---

## Pagination vs Infinite Scroll

| 模式 | 適用情境 |
|------|----------|
| **Pagination** | 有目標搜尋、需要定位、SEO |
| **Infinite Scroll** | 瀏覽探索、社交媒體、圖片庫 |

---

## 回饋類型

| 動作 | 回饋類型 | 範例 |
|------|----------|------|
| 點擊按鈕 | 視覺變化 | 按鈕按下效果 |
| 提交表單 | 載入狀態 | Spinner + "處理中..." |
| 操作成功 | 成功訊息 | Toast: "已儲存" |
| 操作失敗 | 錯誤訊息 | 紅色提示 + 解決方法 |
| 長時間等待 | 進度指示 | 進度條 + 預估時間 |
