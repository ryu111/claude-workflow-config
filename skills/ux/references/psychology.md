# UX 心理學法則

運用認知心理學原理，設計更符合人類思維方式的介面。

---

## 1. Jakob's Law 雅各布定律

> **使用者會將從其他網站累積的經驗期望帶到你的網站。**

### 原則

使用者花費大部分時間在「其他」網站上，因此他們期望你的網站運作方式與其他熟悉的網站相同。

### 應用

```
✓ 遵循常見的 UI 模式
✓ 將重要元素放在預期位置
✓ 使用熟悉的圖示和標籤
✓ 保持與競品的基本一致性
```

### 範例

```
常見網站佈局期望：

┌─────────────────────────────────┐
│ [Logo]      [導航]    [搜尋][👤]│  ← Header
├─────────────────────────────────┤
│                                 │
│     主要內容區域                 │  ← Main
│                                 │
├─────────────────────────────────┤
│     頁尾連結                     │  ← Footer
└─────────────────────────────────┘

❌ 把 Logo 放右邊、導航放底部 → 違反預期
✅ 遵循慣例，減少學習成本
```

### 何時打破慣例

只有當：
1. 創新帶來顯著價值
2. 有足夠的引導幫助使用者適應
3. 經過使用者測試驗證

## 2. Fitts's Law 費茲定律

> **移動到目標的時間取決於目標的距離和大小。**

### 公式概念

```
時間 = a + b × log₂(距離/大小 + 1)

→ 目標越大、越近 = 越容易點擊
```

### 應用

| 原則 | 實作 |
|------|------|
| 大目標 | 主要按鈕 ≥ 44px |
| 近距離 | 相關操作靠近 |
| 邊緣優勢 | 利用螢幕邊緣（無限大） |
| 群組化 | 相關按鈕放一起 |

### 範例

```
❌ 難以點擊
[ x ] ← 10px 的關閉按鈕

✅ 容易點擊
[  X  ] ← 44px 的關閉按鈕

❌ 危險操作容易誤觸
[儲存] [刪除] ← 緊鄰，容易誤點

✅ 安全設計
[儲存]           [刪除] ← 間隔距離
```

### 觸控裝置

```css
/* 最小觸控區域 */
.touch-target {
  min-width: 44px;
  min-height: 44px;
  /* 或使用 padding 擴大可點擊範圍 */
}
```

## 3. Hick's Law 希克定律

> **做決定的時間隨著選項數量和複雜度增加而增加。**

### 公式概念

```
反應時間 = a + b × log₂(n + 1)

n = 選項數量
```

### 應用

```
✓ 減少選項數量
✓ 將選項分類分組
✓ 漸進式揭露
✓ 提供推薦/預設
```

### 範例

```
❌ 選項過多
[選項1] [選項2] [選項3] [選項4] [選項5]
[選項6] [選項7] [選項8] [選項9] [選項10]
↑ 使用者決策癱瘓

✅ 簡化選項
[推薦方案 ⭐]
[檢視全部選項 ▼]
↑ 先提供推薦，需要時再展開

✅ 分類組織
主餐：[選項1] [選項2] [選項3]
飲料：[選項A] [選項B]
甜點：[選項X] [選項Y]
↑ 分類減少認知負擔
```

### 導航設計

```
❌ 扁平導航（7+ 項目）
[首頁] [產品] [服務] [案例] [團隊] [部落格] [關於] [聯繫] [FAQ]

✅ 分層導航
[首頁] [產品 ▼] [關於 ▼] [聯繫]
         ├ 產品A
         ├ 產品B
         └ 全部產品
```

## 4. Miller's Law 米勒定律

> **普通人短期記憶只能容納 7 ± 2 個項目。**

### 應用

| 情境 | 建議 |
|------|------|
| 導航項目 | 5-7 個主要項目 |
| 表單分組 | 每組 3-5 個欄位 |
| 清單顯示 | 分頁或虛擬捲動 |
| 步驟流程 | 3-5 個步驟 |

### 範例

```
❌ 一次顯示太多
步驟：1 → 2 → 3 → 4 → 5 → 6 → 7 → 8 → 9 → 完成
↑ 使用者感到壓力

✅ 分塊處理
大步驟 1: 基本資訊 (含 3 小步驟)
大步驟 2: 付款資訊 (含 2 小步驟)
大步驟 3: 確認送出 (含 1 步驟)
↑ 3 個大步驟，更易管理
```

### 電話號碼分塊

```
❌ 0912345678 ← 10 位數字，難記憶
✅ 0912-345-678 ← 分成 3 塊，易記憶
```

## 5. Tesler's Law 泰斯勒定律（複雜性守恆）

> **任何系統都存在一定的複雜性，無法被消除。**

### 原則

- 複雜性只能轉移，不能消除
- 設計師應承擔複雜性，而非使用者
- 過度簡化可能損害功能

### 應用

```
複雜性可以轉移給：
1. 設計師 → 設計更聰明的介面
2. 開發者 → 寫更複雜的程式碼
3. 使用者 → （應避免）

✅ 地址自動填入
使用者輸入：台北市
系統處理：自動帶出區域選項
↑ 複雜性由系統承擔

✅ 智慧預設
根據使用者地區自動設定：
- 時區
- 語言
- 貨幣
↑ 減少使用者需要做的決定
```

### 警告

```
❌ 過度簡化
移除專業用戶需要的進階功能
↑ 簡化不等於移除功能

✅ 漸進式複雜度
基本模式：簡單介面
進階模式：完整功能
↑ 根據使用者程度提供適當複雜度
```

## 6. Von Restorff Effect 隔離效應

> **當多個相似物件存在時，與眾不同的那個最容易被記住。**

### 應用

```
✓ 主要 CTA 按鈕突出
✓ 重要資訊視覺差異化
✓ 錯誤訊息明顯標示
✓ 新功能標籤
```

### 範例

```
❌ 所有按鈕相同
[取消] [儲存] [送出]
↑ 無法快速識別主要動作

✅ 主要動作突出
[取消]  [儲存]  [送出 ★]
 灰色    灰色    藍色填滿
↑ 主要動作一眼可見
```

### 定價頁面

```
┌──────────┬──────────┬──────────┐
│  基本    │ 專業 ⭐  │  企業    │
│  $9/月   │ $29/月   │  $99/月  │
│          │ 最受歡迎  │          │
│ [選擇]   │ [選擇]   │ [聯繫]   │
└──────────┴──────────┴──────────┘
             ↑ 視覺突出
```

### 注意事項

```
❌ 過度使用
如果所有東西都突出，就沒有東西突出

✅ 策略性使用
只突出最重要的 1-2 個元素
```

## 7. Zeigarnik Effect 蔡格尼克效應

> **人們對於未完成的任務記憶比已完成的更深刻。**

### 應用

```
✓ 進度指示器
✓ 任務清單
✓ 成就系統
✓ 儲存草稿
```

### 範例

```
個人檔案完成度
┌─────────────────────────────────┐
│ 你的檔案已完成 65%              │
│ ████████████░░░░░░░░░░ 65%      │
│                                 │
│ 還差一點就完成了！               │
│ ☑ 基本資料                      │
│ ☑ 頭像                          │
│ ☐ 自我介紹 ← 點擊完成           │
│ ☐ 連結社群帳號                   │
└─────────────────────────────────┘
↑ 未完成的任務產生完成動機
```

### 學習平台

```
課程進度
├── ✅ 第一章：入門 (已完成)
├── ✅ 第二章：基礎 (已完成)
├── 🔵 第三章：進階 (進行中 - 60%)
├── ⚪ 第四章：實戰
└── ⚪ 第五章：精通

↑ 看到進行中的任務，促使繼續
```

## 8. Peak-End Rule 峰終定律

> **人們對體驗的記憶主要基於「高峰時刻」和「結束時刻」的感受。**

### 應用

```
✓ 創造正面高峰時刻
✓ 設計愉悅的結束體驗
✓ 避免負面峰值
✓ 用正面結尾彌補過程不順
```

### 範例

```
電商結帳流程：

過程可能有點繁瑣...
但最後：
┌─────────────────────────────────┐
│  🎉 恭喜！訂單已完成             │
│                                 │
│  您的訂單將於 3 天內送達         │
│                                 │
│  訂單編號：#12345               │
│  [追蹤訂單]                      │
│                                 │
│  🎁 專屬優惠：下次購物享 9 折    │
└─────────────────────────────────┘
↑ 正面的結束體驗
```

### 應用場景

| 時機 | 設計 |
|------|------|
| 任務完成 | 慶祝動畫、感謝訊息 |
| 成就達成 | 勳章、里程碑 |
| 關閉應用 | 友善的告別訊息 |
| 退款/取消 | 體貼的處理流程 |

## 9. Serial Position Effect 序列位置效應

> **人們最容易記住列表中的第一個（首因效應）和最後一個（近因效應）項目。**

### 應用

```
✓ 重要資訊放開頭或結尾
✓ 主要 CTA 放列表首/尾
✓ 導航設計考慮位置
```

### 範例

```
導航列：
[首頁] [產品] [服務] [關於] [聯繫]
  ↑                          ↑
最易記住              最易記住

表單摘要：
┌─────────────────────────────────┐
│ 訂單摘要                        │  ← 第一項：重要
│ ├ 商品明細                      │
│ ├ 運費                          │
│ ├ 折扣                          │
│ └ 總金額：$1,234               │  ← 最後項：重要
└─────────────────────────────────┘
```

## 10. Gestalt Principles 格式塔原則

> **人類視覺傾向將元素組織成有意義的整體。**

### 主要原則

#### 接近性 Proximity
```
相近的元素被視為一組：

✅ 分組清晰
[標籤 1] [輸入框 1]

[標籤 2] [輸入框 2]

❌ 分組不清
[標籤 1]
[輸入框 1]
[標籤 2]
[輸入框 2]
```

#### 相似性 Similarity
```
相似的元素被視為一組：

[主按鈕] [主按鈕] [主按鈕]  ← 藍色
[次按鈕] [次按鈕]          ← 灰色

↑ 顏色相同的被視為同類功能
```

#### 閉合性 Closure
```
人腦會自動補完不完整的形狀：

 ____
|    |   ← 雖然不完整，但能辨識為方形
|____|
```

#### 連續性 Continuity
```
視線沿著路徑流動：

[步驟1] ──→ [步驟2] ──→ [步驟3]
↑ 連續的線條引導視線
```

#### 共同區域 Common Region
```
在同一區域內的元素被視為一組：

┌─────────────┐  ┌─────────────┐
│ 個人資訊     │  │ 付款資訊     │
│ [姓名]      │  │ [卡號]      │
│ [Email]    │  │ [到期日]    │
└─────────────┘  └─────────────┘
↑ 邊框創造分組
```

## 11. Cognitive Load Theory 認知負荷理論

> **減少不必要的心智負擔，保留認知資源給重要任務。**

### 認知負荷類型

| 類型 | 說明 | 設計目標 |
|------|------|----------|
| 內在負荷 | 任務本身複雜度 | 分解任務 |
| 外在負荷 | 介面造成的負擔 | 最小化 |
| 相關負荷 | 學習理解的投入 | 最佳化 |

### 減少策略

```
1. 分塊呈現 - 每次 3-5 項
2. 視覺層級 - 突出重要資訊
3. 漸進揭露 - 按需顯示
4. 一致性 - 減少學習成本
5. 外部記憶 - 顯示而非記憶
```

## Checklist

### 學習成本
- [ ] 遵循使用者熟悉的模式
- [ ] 使用標準圖示和術語
- [ ] 與平台慣例一致

### 決策效率
- [ ] 選項數量合理 (5-7)
- [ ] 提供推薦/預設選項
- [ ] 分類組織複雜選項

### 視覺設計
- [ ] 點擊目標夠大 (≥44px)
- [ ] 重要元素視覺突出
- [ ] 使用分組和層級
- [ ] 資訊分塊呈現

### 體驗設計
- [ ] 顯示進度和狀態
- [ ] 設計愉悅的完成時刻
- [ ] 系統承擔複雜性
