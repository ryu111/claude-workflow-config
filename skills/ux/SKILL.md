# UX 使用者體驗設計規範

確保產品易用、直覺且符合使用者需求。

## 設計思維

### 1. 理解使用者

| 問題 | 說明 |
|------|------|
| 誰是使用者？ | 目標用戶特徵 |
| 目標是什麼？ | 想完成什麼任務 |
| 痛點是什麼？ | 現有方案問題 |
| 使用情境？ | 環境、裝置 |

### 2. 使用者旅程

```
[進入點] → [認知] → [決策] → [行動] → [完成/回饋]
```

## 核心原則

### 1. 認知負荷最小化

```
❌ 一次顯示 20 個欄位
✅ 分成 3-4 步驟，每步 5-6 欄位
```

- 分塊呈現（每次 3-5 項）
- 漸進式揭露
- 使用熟悉模式

### 2. 清晰視覺層級

```
1. 標題（最大、最醒目）
2. 主要按鈕（對比色）
3. 關鍵資訊（粗體）
4. 次要內容
5. 輔助資訊（較小）
```

### 3. 即時回饋

每個動作都需要回饋。

### 4. 錯誤預防

```
❌ "錯誤：無效輸入"
✅ "電話格式不正確。請輸入 10 位數字，如：0912345678"
```

### 5. 使用者控制

- 可逆操作（Undo）
- 確認破壞性操作
- 記住偏好
- 允許跳過

## Checklist

### 流程
- [ ] 目標明確
- [ ] 步驟最少
- [ ] 可返回上一步
- [ ] 有完成狀態

### 回饋
- [ ] 每動作有回饋
- [ ] 載入狀態清楚
- [ ] 錯誤訊息有幫助

### 導航
- [ ] 知道在哪裡
- [ ] 知道可去哪
- [ ] 可回起點

### 容錯
- [ ] 重要操作有確認
- [ ] 可復原
- [ ] 錯誤可修正
