# UX 使用者體驗設計規範

確保產品易用、直覺且符合使用者需求。

## 設計思維

### 1. 理解使用者

| 問題 | 說明 |
|------|------|
| 誰是使用者？ | 目標用戶特徵 |
| 目標是什麼？ | 想完成什麼任務 |
| 痛點是什麼？ | 現有方案問題 |
| 使用情境？ | 環境、裝置 |

### 2. 使用者旅程

```
[進入點] → [認知] → [決策] → [行動] → [完成/回饋]
```

---

## 核心原則

### 1. 認知負荷最小化

```
❌ 一次顯示 20 個欄位
✅ 分成 3-4 步驟，每步 5-6 欄位
```

- 分塊呈現（每次 3-5 項）
- 漸進式揭露
- 使用熟悉模式

### 2. 清晰視覺層級

```
1. 標題（最大、最醒目）
2. 主要按鈕（對比色）
3. 關鍵資訊（粗體）
4. 次要內容
5. 輔助資訊（較小）
```

### 3. 即時回饋

每個動作都需要回饋：
- < 100ms：感覺即時
- 100-1000ms：顯示狀態變化
- > 1s：進度指示器

### 4. 錯誤預防與處理

```
❌ "錯誤：無效輸入"
✅ "電話格式不正確。請輸入 10 位數字，如：0912345678"
```

### 5. 使用者控制

- 可逆操作（Undo）
- 確認破壞性操作
- 記住偏好
- 允許跳過

## 心理學法則速查

| 法則 | 核心概念 | 應用 |
|------|----------|------|
| Jakob's Law | 熟悉性 | 遵循常見 UI 模式 |
| Fitts's Law | 大+近=易點 | 大按鈕、近距離 |
| Hick's Law | 選項少=決策快 | 減少選項數量 |
| Miller's Law | 7±2 限制 | 分塊呈現資訊 |
| Zeigarnik | 未完成更記憶 | 進度指示器 |
| Peak-End | 峰值+結尾 | 設計好的結束體驗 |

詳細說明 → `references/psychology.md`

## 情感設計

### Don Norman 三層次

| 層次 | 關注點 | 設計要點 |
|------|--------|----------|
| Visceral 本能層 | 第一印象 | 視覺美感、動畫 |
| Behavioral 行為層 | 使用體驗 | 功能、效率、回饋 |
| Reflective 反思層 | 意義 | 品牌、自我形象 |

## 微互動設計

### 四要素

```
Trigger → Rules → Feedback → Loops
觸發器 → 規則  → 回饋    → 循環
```

### 時間規範

| 類型 | 時間 |
|------|------|
| Hover 效果 | 150ms |
| 按鈕點擊 | 100-200ms |
| Modal 開啟 | 200-300ms |
| 頁面轉場 | 300-400ms |

## Checklist

### 流程
- [ ] 目標明確
- [ ] 步驟最少
- [ ] 可返回上一步
- [ ] 有完成狀態

### 回饋
- [ ] 每動作有回饋
- [ ] 載入狀態清楚
- [ ] 錯誤訊息有幫助

### 導航
- [ ] 知道在哪裡
- [ ] 知道可去哪
- [ ] 可回起點

### 容錯
- [ ] 重要操作有確認
- [ ] 可復原
- [ ] 錯誤可修正

### 無障礙
- [ ] 可鍵盤操作
- [ ] 對比度足夠
- [ ] 表單有標籤
