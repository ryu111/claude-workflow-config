# 多風格設計提案模板

產生多個風格方案供用戶選擇時使用此模板。

---

## 使用說明

1. 收集用戶需求（產品類型、受眾、品牌調性）
2. 根據 `styles/industry.md` 選擇候選風格
3. 填入此模板產生 3-5 個方案
4. 提供比較表幫助決策

## 方案比較

| 維度 | 方案一 | 方案二 | 方案三 |
|------|--------|--------|--------|
| **現代感** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **專業度** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| **差異化** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| **親和力** | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **開發難度** | 中 | 低 | 高 |
| **無障礙** | 良好 | 優秀 | 需調整 |
| **效能** | 優秀 | 優秀 | 需優化 |

### 維度說明

- **現代感**：視覺風格是否符合當前趨勢
- **專業度**：是否傳達專業、可信賴的形象
- **差異化**：與競品的區隔程度
- **親和力**：對目標受眾的友善程度
- **開發難度**：前端實現的複雜度
- **無障礙**：WCAG 合規程度
- **效能**：對頁面載入/渲染的影響

## 填寫指南

### 風格定位
用一句話說明選擇理由，例如：
- "Glassmorphism 帶來現代感和高端質感，符合 AI 產品的創新形象"
- "Corporate Clean 傳達專業可靠，符合金融產品的信任需求"

### 色彩方案
- 從 `palettes.md` 選擇預設方案
- 或根據品牌色自行調整
- 確保對比度符合 WCAG AA（4.5:1）

### 字體配對
- 從 `font-pairs.md` 選擇
- 確保包含 Google Fonts 連結
- 考慮中文字體需求

### 範例佈局
- 用 ASCII 繪製簡單框線圖
- 標註主要區塊位置
- 不需要太詳細，表達結構即可

### 方案比較
- 誠實評估各方案優缺點
- 評分要有區別，不要全給滿分
- 根據用戶需求調整權重
